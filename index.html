  • Automatic scaling
  • Almost no configuration
  • Automatic updates
  • No system administration
  • No performance tuning (almost)
  • Load balancing

App Engine Instances

How App Engine scales

App Engine Instances

  • Instances are the basic building blocks of App Engine
  • Language runtime
  • The App Engine APIs
  • Your application's code
  • Memory (128MB - 1024MB)
  • Start up and shut down automatically based on current needs
  • Resident instances are also supported

Instance types

Frontend Instances:

  • Running your code
  • Handle all requests by default, including task queues and cron jobs
  • Scaling dynamically based on the incoming requests
  • Limited in how long a request can run (60 seconds - 10 minutes)

Backend Instances:

  • An instance whose live length is determined by your configuration
  • Limited scaling based on your settings
  • You can configure backends to work with task queues and cron jobs

App Engine High Level View

App Engine Services

  • Datastore API - distributed NoSQL database based on BigTable
  • App Identity API - identify itself and to assert its identity to external systems
  • Blobstore API - serve data objects that are much larger than the size allowed for objects in the Datastore service
  • Capabilities API - detect outages and scheduled downtime
  • Channel API - creates a persistent connection between your JavaScript application and Google servers
  • Google Cloud Endpoints - easy way to create API endpoints
  • Immages API - image manipulation API
  • Logs API - provides access to the application and request logs for your application
  • Mail API - send and receive emails
  • Memcache API - distributed in-memory data cache

App Engine Services

  • Multitenancy and the Namespaces API - multitenancy in your applications
  • OAuth
  • Prospective Search
  • Search - full text search
  • Task Queues API - perform work initiated by a user request, outside of that request
  • URL Fetch - communicate with other hosts over the Internet using HTTP and HTTPS requests
  • Users API - authenticate users who have Google Accounts or OpenID identifiers
  • XMPP API - send and receive instant messages to and from users of XMPP-compatible instant message services

App Engine High Level View

App Engine Restrictions

You need these to scale!

Python VM restrictions

  • It is a "pure" Python environment
  • Cannot load Python modules with C code
  • App cannot write data to the local file system
  • App cannot make arbitrary network connections
  • Interpreter raises an exception when an app attempts to import a module from the standard library known to not work within the sandbox restrictions

HTTP request restrictions

  • Each incoming HTTP request can be no larger than 32MB
  • Maximum response size 32MB
  • A request handler has 60 seconds to generate and return a response
  • A task must finish executing and send an HTTP response value between 200–299 within 10 minutes of the original request
  • App Engine is optimized for applications with short-lived requests
    (few hundred milliseconds)

Datastore restrictions

  • No joins
  • No inequality filtering on multiple properties
  • No aggregate, grouping queries on large datasets
  • No filtering of data based on results of a subquery
  • All queries on App Engine are served by pre-built indexes
  • Maximum entity size: 1MB

App Engine Datastore

Datastore

  • Distributed partitioned Datastore (BigTable, NoSQL)
  • Schema less
  • Horizontal staling
  • Replicated and fault tolerant
  • Parallel / async processing
  • No joins - but there are model relations
  • Optimized for reading

Datastore → RDBMS

Terminology

Table → Kinds

Row → Entity

Column → Property

Datastore Entity

Objects in the App Engine Datastore are known as entities

  • An entity has one or more named properties
  • Each of which can have one or more values
  • Each entity is of a particular kind
  • Each entity is identified by key

Datastore Entity Key

The key consists of the following components:

  • The entity's kind
  • An identifier, which can be either: string (key name) or numeric ID
  • An optional ancestor path locating the entity within the Datastore hierarchy

Datastore Model - Define

from google.appengine.ext import ndb

class User(ndb.Model):
  username = ndb.StringProperty(required=True)
  password = ndb.StringProperty(required=True)
  bio = ndb.TextProperty(default=None)
  attributes = ndb.StringProperty(repeated=True)
  attribute_count =
    ndb.ComputedProperty(lambda self: len(self.attributes))
  gender =
    ndb.StringProperty(default='u', choices=set(['u', 'm', 'f']))
  created = ndb.DateTimeProperty(auto_now_add=True)
  updated = ndb.DateTimeProperty(auto_now=True)

Memcache

Distributed, fast in-memory cache

from google.appengine.api import memcache

def get_data(key):
  data = memcache.get(key)
  if data is not None:
    return data
  else:
    data = self.query_for_data(key)
    memcache.add(key, data, 60)
  return data
